Введення в CSS

CSS (Cascading Style Sheets, каскадні таблиці стилей) - це мова для опису і зміни зовньошнього вигляду елементів. HTML використовується для визначення певної структури та семантики вмісту документа, а CSS для його оформлення та позиціонування.

Синтаксис

Блок CSS-коду називається правилом, яке складається з селектора та групи оголошень у фігурних дужках.

Селектор вказує браузеру на те, до яких елементів потрібно застосувати стилі цього правила. Кожне оголошення складається з пари властивість: значення; де властивість і його значення поділяються двокрапкою з пробілом і обов'язково закінчуються точкою з комою. Вони описують, що саме потрібно зробити зі стилями елемента. 

                    
        селектор {
            властивість: значення;
            властивість: значення;
            властивість: значення;
        }

Підключення стилей

Є три способи додати стилі до HTML-документу: вбудовані стилі (inline styles), вбудована таблиця стилів (embedded stylesheet) та зовнішня таблиця стилів (external stylesheet). Кожен спосіб має свої переваги, недоліки, обмеження і сферу застосування.

Вбудовані стилі

Вбудовані (інлайн) стилі задаються тегом прямо в атрибуті style. Їх неможливо масштабувати, складно перевизначати та використовувати повторно, тому вони застосовуються в дуже поодиноких випадках. Наприклад, для динамічних стилів, які встановлюються під час виконання JavaScript коду, коли ми заздалегідь не знаємо, яке буде значення властивості. 

                    
        <p style="color:blue; font-size:24px;">Этот текст будет синим.</p>

Вбудована таблиця стилів

При такому підході CSS-правила додаються всередину тега <style> у шапці документа. Їх неможливо використовувати повторно на інших сторінках і складно масштабувати та підтримувати. 

                    
        <head>
        <style type="text/css">
            p {
                color: blue;
                font-size: 24px;
            }
        </style>
        </head>

Цікаво

Іноді цей підхід використовується для оптимізації швидкості відображення сторінки, додаючи найважливіші стилі прямо до HTML-документу. Ця просунута техніка називається Critical CSS.

Зовнішня таблиця стилів

При такому підході CSS-правила додаються всередину тега <style> у шапці документа. Їх неможливо використовувати повторно на інших сторінках і складно масштабувати та підтримувати. 

                    
        <head>
        <link rel="stylesheet" href="./css/styles.css"/>
        </head>
  • На одному рівні з index.html створюється папка css, а в ній файл стилів styles.css.
  • У тезі <head/>, використовуючи тег <link/>, раніше створений файл підключається.
  • В атрибуті href вказується шлях до файлу стилів щодо HTML-документу.
  • В атрибуті rel вказується тип документа, що підключається - stylesheet (таблиця стилів).

Корисно

Можна підключити скільки завгодно файлів CSS, додаючи для кожного тег <link>.

Селектори

За допомогою різних селекторів можна вибрати один чи групу елементів. Селекторів досить багато, тому розглянемо основні, які можуть застосовуватися щодня.

Селектор елемента (тега)

Описує, до якого типу елементів буде застосовано CSS-правило. Селектор елемента недостатньо специфічний для того, щоб стилізувати окремі елементи, тому найчастіше застосовується для оформлення всіх тегів одного типу на сторінці. Наприклад, якщо у всіх посилань у документі потрібно прибрати підкреслення.

                        
        /* Застосується до всіх абзаців на сторінці */
        p {
            font-size: 24px;
        }
                            
        /* Застосується до всіх посилань на сторінці */
        a {
            text-decoration: none;
        }

Селектор класу

Основний селектор у сучасній розробці. Використовується у зв'язку з глобальним атрибутом class. Ім'я класу задається тільки англійською мовою, і це обов'язково іменник, що описує, що це за елемент. Для поповнення словникового запасу використовуйте довідник, словник термінів та скорочення. 

                        
        <h1 class="title">Привіт, я Mango.</h1>
        <p class="text">
            Ласкаво просимо на мою особисту сторінку. Тут можна подивитись
            <a class="link" href="">проєкти</a>.
        </p>

У CSS використовується селектор класу, щоб встановити стилі для одного і більше елементів з однаковим значенням атрибута class (ім'ям класу). У селекторі перед ім'ям класу ставиться точка .. 

                        
        /* Застосується до всіх тегів з класом title */
        .title {
            font-weight: 500;
        }
                            
        /* Застосується до всіх тегів з класом text */
        .text {
            color: brown;
            font-size: 18px;
        }
                            
        /* Застосується до всіх тегів з класом link */
        .link {
            text-decoration: none;
        }

Увага

В імені класу використовуються лише маленькі літери та тире. Наприклад, list, list-item, logo-image і т. д. Підкреслення list_item, великі літери ListItem або цифри listitem27 вважаються поганим тоном.

Композиція класів

У елемента може бути кілька класів, тоді вони записуються через пропуск. 

                        
        <element class="value1 value2 value3"></element>

Це можна використовувати для комбінування та повторного використання стилів. Створимо розмітку та стилі для компонента оповіщень. 

                        
        <p class="alert success">Поповнення рахунку виконано</p>
        <p class="alert warning">Увага, будуть змінені тарифи</p>
        <p class="alert error">Помилка проведення транзакції</p>

У стилях описуємо кілька правил, загальні всім сповіщень і специфічні кожного типу. 

                        
        /* Загальні стилі для всіх типів сповіщень */
        .alert {
            font-size: 24px;
            font-weight: 500;
        }
                            
        /* Специфічні стилі для кожного типу */
        .success {
            color: green;
        }
                            
        .warning {
            color: orange;
        }
                            
        .error {
            color: red;
        }

Селектор ідентифікатора

Як і з класом, теги можна задавати глобальний атрибут id. Відмінність у тому, що значення id має бути унікальним на сторінці, тобто не може бути два елементи з однаковими значеннями ідентифікатора. 

                        
        <h1 id="title">Заголовок сторінки</h1>

В CSS-селекторі, перед ім'я ідентифікатора, ставиться символ решітки #.

                        
        #title {
            font-weight: 500;
            color: orange;
        }

Корисно

Не використовуйте ідентифікатори для оформлення. Правило унікальності значення унеможливлює повторне використання стилів. Ідентифікатори призначені для «якорів».

Селектор нащадка

Селектор нащадка, або контекстний селектор, застосовує стилі до дітей та нащадків елемента за будь-якої глибини вкладеності. 

                    
        <a href="">Читайте наш блог!</a>

        <ul class="social-links">
            <li>
                <a href="">Twitter</a>
            </li>
            <li>
                <a href="">Instagram</a>
            </li>
            <li>
                <a href="">Facebook</a>
            </li>
        </ul>

Наприклад, замість того, щоб задавати стиль усім посиланням, потрібно змінити стилі тільки тих, які знаходяться всередині списку з класом social-links. 

                    
        /* ❌ Застосується до всіх посилань в документі */
        a {
            color: blue;
            text-decoration: none;
        }
                            
        /* ✅ Застосується тільки до посилань, які лежать всередині тега з класом social-links */
        .social-links a {
            color: blue;
            text-decoration: none;
        }

Селектор дитини

Ще один розповсюджений селектор, який дозволяє обрати тільки безпосередню дитину всередині батьківського елемента. Використаємо розмітку для магаторівневого меню магазину, в якому необхідно стилізувати теги <li> у списку з класом menu, але не submenu.

                    
        <ul class="menu">
            <li>
                Комп'ютери та комплектуючі
                <ul class="submenu">
                    <li>Процессоры</li>
                    <li>Мониторы</li>
                    <li>Видеокарты</li>
                </ul>
            </li>
            <li>
                Побутова техніка
                <ul class="submenu">
                    <li>Холодильники</li>
                    <li>Телевізори</li>
                    <li>Пральні машини</li>
                </ul>
            </li>
            <li>
                Товари для дому
                <ul class="submenu">
                    <li>Крісла</li>
                    <li>Матраци</li>
                    <li>Електрокаміни</li>
                </ul>
            </li>
        </ul>

Якщо використовувати селектор нащадка (контекстний), то CSS-правило застосується до всіх тегів <li>, і червона рамка буде у кожного.

                    
        /* ❌ Не вирішує задачу */
        .menu li {
            border: 1px solid tomato;
        }

Селектор дитини допомагає вирішити цю проблему, обравши тільки ті <li>, які є дітьми списку ul.menu.

                    
        /* ✅ Те що треба */
        .menu > li {
            border: 1px solid tomato;
        }

Селектор атрибута

Дозволяє застосувати стилі до елементів із певним атрибутом або його значенням.

елемент[атрибут]

Застосує стилі до всіх елементів, які мають вказаний атрибут. 

                    
        <a href="https://htmlreference.io" title="Довідник тегів">
            htmlreference.io
        </a>
                          
        <a href="https://cssreference.io" title="Довідник стилей">
            cssreference.io
        </a>
                          
        <a href="https://www.youtube.com">YouTube </a>

Обере всі посилання з атрибутом title.

                    
        a[title] {
            color:teal;
        }

елемент[атрибут="значення"]

Застосує стилі до всіх елементів, у яких є атрибут з вказаним значенням.

                    
        <a href="https://htmlreference.io" title="Довідник тегів">
            htmlreference.io
        </a>
                          
        <a href="https://cssreference.io" title="Довідник стилей">
            cssreference.io
        </a>
                          
        <a href="https://www.youtube.com">YouTube </a>

Обере всі посилання з атрибутом href і відповідним значенням.

                    
        /* Незвичне форматування фігурних дужок це нормально */
        a[href="https://htmlreference.io"]
        {
            color: teal;
        }
                        
        a[href="https://cssreference.io"]
        {
            color: orange;
        }
                        
        a[href="https://www.youtube.com"]
        {
            color: tomato;
        }

Псевдокласи стану

Селектори стану використовуються для застосування стилей до інтерактивних елементів при певній події, наприклад наведення курсора на посилання або фокусування з клавіатурі.

Псевдоклас визначає конкретний стан елемента та прикріплений до селектора тегу, класу і т.д. Пробіла між селектором та псевдокласом немає, щоб показати, що вони пов'язані один з одним. Якщо додати пробіл, то псевдоклас застосується до всіх елементів в документі.

                    
        селектор:псевдоклас {
            /* Властивість */
        }

Псевдоклас :hover

Активується коли курсор миші знаходиться у межах елемента, наприклад при наведенні миші на посилання або будь-який інший елемент.

                    
        <p> 
            Чудову шпаргалку по тегам можна знайти
            <a class="link" href="https://htmlreference.io"> по ссылке</a>
        </p>

Додамо посиланню базовий стиль та стиль при наведенні.

                    
        .link {
            color: teal;
        }
                          
        .link:hover {
            color: tomato;
        }

Псевдокласи можна застосовувати до будь-яких селекторів, в тому ж числі контекстних.

                    
        <ul class="social-links">
            <li><a class="link" href="https://twitter.com">Twitter</a></li>
            <li><a class="link" href="https://www.instagram.com">Instagram</a></li>
            <li><a class="link" href="https://www.facebook.com">Facebook</a></li>
        </ul>

Додамо посиланню всередні списку базовий стиль та стиль при наведенні.

                    
        .social-links .link {
            color: teal;
        }
                          
        .social-links .link:hover {
            color: tomato;
        }

Корисно

Такий селектор читається як "При ховері по елементам з класом link, які знаходяться всередині елементів з класом social-links". Тобто читаються селектори справа наліво.

Псевдоклас :focus

Активується коли інтерактивний елемент (посилання, кнопка, поле форми) отримує фокус при навігації по сторінці клавіатурою (клавішею Tab).

                        
        <ul class="social-links">
            <li><a class="link" href="https://twitter.com">Twitter</a></li>
            <li><a class="link" href="https://www.instagram.com">Instagram</a></li>
            <li><a class="link" href="https://www.facebook.com">Facebook</a></li>
        </ul>

Додамо посиланню всередині списку базовий стиль та стиль при фокусі.

                        
        .social-links .link {
            color: teal;
        }
                              
        .social-links .link:focus {
            color: orange;
        }

Для того, щоб врахувати наведення та фокусування, варто стилізувати одразу обидка стани. Це робиться простим перерахуванням селекторів.

                    
        .social-links .link {
            color: teal;
        }
                                  
        /* Два та більше селекторів відокремлюються комою */
        .social-links .link:hover,
        .social-links .link:focus {
            color: orange;
        }

Псевдоклас :active

Відбувається при активації елемента. Наприклад, посилання стає активним, якщо навести на неї курсор та клацнути мишкою. Не дивлячись на те, що активним може стати практично будь-який елемент, псевдоклас :active використовується в основному для посилань та кнопок.

                    
        <ul class="social-links">
            <li><a class="link" href="https://twitter.com">Twitter</a></li>
            <li><a class="link" href="https://www.instagram.com">Instagram</a></li>
            <li><a class="link" href="https://www.facebook.com">Facebook</a></li>
        </ul>       
                    
                
                    
        .social-links .link {
            color: teal;
        }
                          
        .social-links .link:hover,
        .social-links .link:focus {
            color: orange;
        }
                          
        .social-links .link:active {
            color: red;
        }

Псевдоклас :visited

Цей псевдоклас активується на посиланнях, які вже були відвідані. За замовчуванням посилання відображаються синім та при відвідуванні стають фіолетовими. 

                    
        <ul class="social-links">
            <li><a class="link" href="https://twitter.com">Twitter</a></li>
            <li><a class="link" href="https://www.instagram.com">Instagram</a></li>
            <li><a class="link" href="https://www.facebook.com">Facebook</a></li>
        </ul>       
                    
                
                    
        .social-links .link {
            color: teal;
        }
                          
        .social-links .link:hover,
        .social-links .link:focus {
            color: orange;
        }
                          
        .social-links .link:active {
            color: red;
        }
                          
        .social-links .link:visited {
            color: green;
        }

Ховер і таблиці

Псевдоклас :hover не обов'язково повинен застосовуватися до посилань, його можна додавати і к іншим елементам документам. Наприклад, до рядків та ячеек таблиці, щоб вони змінювали колір фону при наведенні курсора миші.

                    
        <table class="schedule">
            <thead>
                <tr>
                    <th>Номер</th>
                    <th>Маршрут</th>
                    <th>Время в пути</th>
                </tr>
            </thead>
            <tbody class="schedule-body">
                <tr>
                    <td>433</td>
                    <td>Киев - Ковель</td>
                    <td>5 часов</td>
                </tr>
                <tr>
                    <td>701</td>
                    <td>Харьков - Днепр</td>
                    <td>7 часов</td>
                </tr>
                <tr>
                    <td>258</td>
                    <td>Львов - Одесса</td>
                    <td>4 часа</td>
                </tr>
            </tbody>
        </table>

Додамо базове оформлення таблиці та стилі для ховера по рядкам.

                    
        .schedule-body > tr:hover {
            background-color: tomato;
            color: white;
        }
                          
        /* Оформлення таблиці */
        .schedule {
            border-collapse: collapse;
        }
                          
        .schedule th,
        .schedule td {
            padding: 8px;
            border: 1px solid #212121;
        }

Властивість cursor

Встановлює тип вказівника миші, коли він знаходиться в межах елемента. Вид курсора залежить від операційної системи та її налаштувань. Змінювати тип курсора просто так погана ідея, це тільки заплутає користувача, коли замість звичної руки або лупи він побачить щось інше. В більшості випадків краще залишити все як є.

Робота з кольором

Колір тексту

За допомогою властивості color можна змінити колір тексту елемента. Сучасні браузери підтримують декілька основних способів опису кольору: ключові слова, rgb і hex.

                    
        <p class="text">
            Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
            veritatis nihil alias iste odit similique sit eius optio veniam, impedit
            cumque fuga facere labore quo id necessitatibus quaerat rerum.
        </p>             
                    
                
                    
        .text {
            color: tomato;
        };

Корисно

На сервисі htmlcolorcodes.com можна подивитись значення кольорів у різних форматах.

Колір фону

Властивість background-color дозволяє змінити колір фона елемента.

                    
        <p class="text">
            Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
            veritatis nihil alias iste odit similique sit eius optio veniam, impedit
            cumque fuga facere labore quo id necessitatibus quaerat rerum.
        </p>             
                    
                
                    
        .text {
            background-color: tomato;
            color: white;
        }

Іменовані кольори

Для опису кольору в стилях можна використовувати набір зарезервованих слів. У сучасній веб-розробці вони вже не використовуються, але бувають корисними якщо треба швидко поставити якийсь колір.

                    
        p {
            color: tomato;
        }

RGB

Є три основні кольори: червоний, зелений та синій. Змішуючи їх у різноманітних пропорціях можна отримати будь-який колір.

У цьому форматі для задання значення використовується функція rgb(), в якій вказується доля кожного з трьох основних кольорів, після чого вони змішуються та виходить потрібний колір.

                        
        rgb(червоний, зелений, синій)

Дапазон значень можна вказувати числами від 0 (повна відсутність кольору) до 255 (максимальна доля). Це основний спосіб задання RGB-кольору.

                    
        /* Чистий червоний колір: 255 червоного, 0 зеленого, 0 синього */
        p {
            color: rgb(255, 0, 0);
        }

Або відсотками від 0% до 100%.

                    
        /* Чистий червоний колір: 100% червоного, 0% зеленого, 0% синього */
        p {
            color: rgb(100%, 0%, 0%);
        }

Увага

Запам'ятовувати комбінації RGB-значень не потрібно, всі кольори є в макеті від дизайнера.

Hexzdecimal

Це опис кольору оснований на записі числа в шістнадцятиричній системи числення. Доля кожного з трьох кольорів описується двома цифрами.

В якості цифр цієї системи числення використовується набір цифр від 0 до 9 та латинські літери від А до F в нижньому та верхньому регістрі.

                        
        /* Чистий чрвоний колір: ff (100%) червоного, 00 (0%) зеленого, 00 (0%) синього */
        p {
            color: #ff0000;
        }
                            
        /* Чистий зелений колір: 00 (0%) червоного, ff (100%) зеленого, 00 (0%) синього */
        p {
            color: #00ff00;
        }
                            
        /* Чистий синій колір: 00 (0%) червоного, 00 (0%) зеленого, ff (100%) синього */
        p {
            color: #0000ff;
        }

Увага

Запам'ятовувати комбінації HEX-значень не потрібно, всі кольори є в макеті від дизайнера.

Прозорість кольору

Основний спосіб додавання прозорості кольору це функція rgba(). На відміну від rgb() ще додається прозорість (альфа канал).

                            
        rgba(червоний, зелений, синій, альфа)

Найчастіше прозорість задається числом від 0 (прозорий) до 1 (непрозорий).

                        
        /* Чистий червоний колір з прозорістю в 30% */
        p {
            background-color: rgba(255, 0, 0, 0.3);
        }

Але можна й відсотками від 0% (прозорий) до 100% (не прозорий).

                        
        /* Чистий червоний колір з прозорістю в 30% */
        p {
            background-color: rgba(100%, 0%, 0%, 30%);
        }

Цікаво

Прозорість кольору можна задати й у Hex-форматі, але через складність такого запису та запам'ятовування значень, завжди використовують функцію rgba().

Каскад стилей

Каскадування - це механізм, який керує кінцевими значеннями властивостей елемента, якщо до нього застосовується декілька CSS-правил.

  • Якщо до елемента застосовується декілька правил, їх властивості комбінуються.
  • Якщо в правилах є однакові властивості з різними значеннями, то вони конфліктують.
                        
        /* Яким буде колір тексту абзаців? */
        p {
            color: blue;
            background-color: orange;
        }
                            
        p {
            color: teal;
        }

Для того, щоб зібрати фінальні стилі елемента та розв'язати конфлікти значень властивостей, браузер використовує два механізма: специфічність та наслідування.

Специфічність селекторів

Для кожного CSS-правила браузер вираховує специфічність (вагу) селектора. Якщо до елемента потрібно застосувати властивості з різних правил, та є конфліктуючі (однакові) властивості, використовується значення властивості з правила з найбільш високою специфічністю селектора.

Значення специфічності складається з чотирьох рангів, важливість яких збільшується справа на ліво.

Селектори 4 рангу

Кожний селектор елемента та псевдоелемента додає одиницю в четвертий ранг.

                    
        <article>
            <h1>Lorem ipsum dolor sit amet</h1>
            <p>
                Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
                veritatis nihil alias iste odit similique sit eius optio veniam, impedit
                cumque fuga facere labore quo id necessitatibus quaerat rerum.
            </p>
            <a href="">Читати далі...</a>
        </article>

Колір тексту абзацу буде помаранчевим, тому що специфічність другого правила вища.

                    
        /* Специфічність - 0 0 0 1 */
        p {
            color: green;
        }
                        
        /* ✅ Специфічність - 0 0 0 2 */
        article > p {
            color: orange;
        }

Селектори 3 рангу

Кожен селектор класу, атрибуту та псевдокласу додає одиницю в третій ранг.

                    
        <article class="post">
            <h1 class="post-title">Lorem ipsum dolor sit amet</h1>
            <p>
                Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
                veritatis nihil alias iste odit similique sit eius optio veniam, impedit
                cumque fuga facere labore quo id necessitatibus quaerat rerum.
            </p>
            <a href="" class="post-link">Читати далі...</a>
        </article>

Колір тексту заголовка буде помаранчевим, тому що специфічність третього правила вища.

                    
        /* Специфічність - 0 0 1 0 */
        .post-title {
            color: green;
        }
            
        /* Специфічність - 0 0 1 1 */
        .post > h1 {
            color: red;
        }
            
        /* ✅ Специфічність - 0 0 2 0 */
        .post > .post-title {
            color: orange;
        }

Колір тексту посилання буде коричневим, тому що специфічність п'ятого правила вища.

                    
        /* Специфічність - 0 0 0 1 */
        a {
            color: green;
        }
                        
        /* Специфічність - 0 0 1 0 */
        .post-link {
            color: orange;
        }
                        
        /* Специфічність - 0 0 1 1 */
        a.post-link {
            color: blue;
        }
                        
        /* Специфічність - 0 0 2 0 */
        .post > .post-link {
            color: red;
        }
                        
        /* ✅ Специфічність - 0 0 2 1 */
        .post > a.post-link {
            color: brown;
        }

Селектори 2 рангу

Кожен селектор ідентифікатора додає одиницю до другого рангу.

                    
        <article class="post">
            <h1 class="post-title id="title"">Lorem ipsum dolor sit amet</h1>
            <p>
                Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
                veritatis nihil alias iste odit similique sit eius optio veniam, impedit
                cumque fuga facere labore quo id necessitatibus quaerat rerum.
            </p>
            <a href="" class="post-link">Читати далі...</a>
        </article>

Колір тексту заголовку буде помаранчевим, тому що специфічність другого правила вища.

                    
        /* Специфічність - 0 0 1 0 */
        .post-title {
            color: green;
        }
                        
        /* ✅ Специфічність - 0 1 0 0 */
        #title {
            color: orange;
        }

Селектори 1 рангу

Вбудовані стилі додають одиницю в перший ранг, мають найвищу специфічність, та перевизначити їх через CSS неможливо.

                    
        <!-- Специфичность такого стиля - 1 0 0 0 -->
        <p style="color: green;">
            Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
            veritatis nihil alias iste odit similique sit eius optio veniam, impedit
            cumque fuga facere labore quo id necessitatibus quaerat rerum.
        </p>

Однакова специфічність

Якщо специфічність однакова, перемагає правило розташоване нижче у коді.

                    
        /* Три селектора елементу з одинаковою специфічністю */
        a {
            color: teal;
        }
                        
        a {
            color: brown;
        }
                        
        /* Так як це правило стоїть нижче, до всіх посилань застосується ораневий */
        a {
            color: orange;
        }
Уникаємо специфічності

Якщо селектор виглядає як A > B > C + X Y, варто замислитись. Чим простіший селектор, тим менше проблем із специфічностю. Вважайте за правило використовувати селектори класу. Атрибут class не тільки допомагає уникнути проблем із специфічністю, але й покращує читабельність HTML-розмітки.

Перший підхід це присвоєння класу загальному батьківському блоку та використання дочірніх селекторів та селекторів нащадка.

                    
            <article class="post">
                <h1>Lorem ipsum dolor sit amet</h1>
                    <p>
                        Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
                        veritatis nihil alias iste odit similique sit eius optio veniam, impedit
                        cumque fuga facere labore quo id necessitatibus quaerat rerum.
                    </p>
                <a href="">Читати далі...</a>
            </article>

Такий CSS гарно працює коли розмітка проста. Але при зростанні важкості розмітки блоку, зручна стилізація по селекторам тегу стає неможливою. Наприклад, якщо у пості буде 5 посилань, а не одне, точково обрати якесь посилання не вийде або селектор буде надто важким.

                    
            .post {
                /* Стилі статі */
            }
                          
            .post > h1 {
                /* Стилі заголовку */
            }
                          
            .post > p {
                /* Стилі тексту */
            }
                          
            .post > a {
                /* Стилі посилання */
            }
                          
            .post > a:hover {
                /* Стилі посилання при ховері */
            }

Другий підхід це додавання класів опису до батьківського та тегів всередині нього.

                    
            <article class="post">
                <h1 class="title">Lorem ipsum dolor sit amet</h1>
                <p class="text">
                    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
                    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
                    cumque fuga facere labore quo id necessitatibus quaerat rerum.
                </p>
                <a class="link" href="">Читати далі...</a>
            </article>

Такий CSS непогано масштабується та підтримується. Це відправна точка для написання хорошого CSS. Тим не менш з зростанням важкості розмітки можуть бути невеликі проблеми зі специфічністю.

                        
            .post {
                /* Стилі статі */
            }
                              
            .post > .title {
                /* Стилі заголовка */
            }
                              
            .post > .text {
                /* Стилі тексту */
            }
                              
            .post > .link {
                /* Стилі посилання */
            }
                              
            .post > .link:hover {
                /* Стилі посилання при ховері */
            }

Третій підход полягає в заданні дуже специфічних класів батьківському блоку та кожному тегу всередині нього за принципом ім'я_батька-ім'я_дитини.

                        
            <article class="post">
                <h1 class="post-title">Lorem ipsum dolor sit amet</h1>
                <p class="post-text">
                    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
                    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
                    cumque fuga facere labore quo id necessitatibus quaerat rerum.
                </p>
                <a href="" class="post-link">Читати далі...</a>
            </article>

При цьому підході використовуються прості селектори класу, не треба думати про специфічність. Такий CSS гарно масштабується та підтримується. У сучасній розробці при написанні стилей використовуються варіації цього підхода.

                            
            .post {
                /* Стилі статі */
            }
                                  
            .post-title {
                /* Стилі заголовка */
            }
                                  
            .post-text {
                /* Стилі тексту */
            }
                                  
            .post-link {
                /* Стилі посилання */
            }
                                  
            .post-link:hover {
                /* Стилі посилання при ховері */
            }

Корисно

Не варто обмежувати себе у виборі лише одного підходу. Усі три методи іменвання розмітки класами та написання стилів можна використовувати разом, в залежності від ситуації.

Наслідування властивостей

Генетичне успадкування звично бачити у житті. Наприклад, у високих батьків найчастіше високі діти, часто передається колір волосся тощо. Щось подібне є й у CSS. Якщо встановити зелений колір тексту батькові, то якщо будь-яке правило не перевизначить це значення, колір тексту всіх його нащадків також буде зеленим.

Спадкування це механізм, з якого значення певних якостей передаються від предка до його нащадкам. Успадковуються, переважно, властивості оформлення тексту, наприклад, колір. Це зручно, тому що не потрібно ставити повторювані стилі для кожного нащадка окремо. 

                        
            <article class="post">
                <h1>Популярное за этот месяц</h1>
                <p>
                    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
                    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
                    cumque fuga facere labore quo id necessitatibus quaerat rerum.
                </p>
                <a href="">Читати далі...</a>
            </article>

Зробимо весь текст поста зеленого кольору.

                        
            .post {
                color: green;
            }

У інструментах розробника на вкладці Styles у самому низу побачимо успадкування для абзацу та заголовка.

У житті не всі атрибути батьків передаються дітям. Те саме і в CSS, не кожна властивість по замовчуванням успадковується нащадками. Фактично, якби всі властивості були успадковані, ефект був би аналогічним до відсутності успадкування взагалі, і довелося б написати дуже багато CSS для скидання стилів успадкованих від батька, щоб перевизначити цю поведінку.

Виключення

Не всі елементи наслідують значення властивостей від нащадків. Наприклад, посилання не наслідує колір тексту.

                            
            <article class="post">
                <h1>Популярное за этот месяц</h1>
                <p>
                    Lorem, ipsum dolor sit amet consectetur adipisicing elit. Obcaecati commodi,
                    veritatis nihil alias iste odit similique sit eius optio veniam, impedit
                    cumque fuga facere labore quo id necessitatibus quaerat rerum.
                </p>
                <a href="" class="post-link">Читати далі...</a>
            </article>

Якщо необхідно, щоб у пості вона була такого ж кольору, як і весь текст, можна використовувати спеціальне значення inherit для властивості color, щоб не дублювати колір у двох місцях. 

                        
            .post {
                color: green;
            }
                                  
            .post-link {
                color: inherit;
            }

Корисно

Значення inherit говорить браузеру у тому, що необхідно успадкувати значення властивості від предка. Для кольору краще використовувати значення currentColor (поточний колір).

Колір тексту сторінки

Щоб задати базовий (домінантний) колір тексту всього документа, створюємо правило для тега <body> і додаємо властивість color. Цей колір буде успадковано всім текстовим контентом, крім посилань. 

                        
            body {
                color: #2a2a2a;
            }

CSS-змінні

Властивості користувача (CSS Custom Properties), також відомі як CSS-змінні - це контейнери для значень, визначені в CSS-документі з метою повторного використання. CSS-змінні спрощують використання та зміну повторюваних значень у файлі стилів. Вони підтримуються браузером і не вимагають ніяких додаткових інструментів. 

                        
                <section class="section">
                    <h1 class="section-title">Section title</h1>
                </section>  
                    
                
                        
            .section {
                /* Інші стилі */
                border: 2px solid #3f51b5;
            }
                              
            .section-title {
                /* Інші стилі */
                color: #3f51b5;
            }
                              
            .section-title::before {
                /* Інші стилі */
                background-color: #3f51b5;
            }

У наведеному прикладі значення #3f51b5 зустрічається тричі. Припустимо це головний колір бренду, який постійно використовується на різних сторінках сайту. Якщо зміниться колір бренду, нам потрібно буде скрізь замінити це значення, а у великому проекті це можуть бути сотні місць. Найкраще, що ми зможемо зробити - використовувати пошук та заміну.

Оголошення змінної

На допомогу приходять CSS-змінні. Ім'я змінної чутливе до регістру і може містити латинські літери, цифри, підкреслення та тире. Створимо змінну --brand-color для зберігання кольору бренду (#3f51b5) і використовуватимемо її. 

                        
            :root {
                --brand-color: #3f51b5;
            }

Глобальні змінні, що доступні в будь-якому іншому правилі, визначаються в селекторі :root - більш специфічному селекторі елемента . Оголошення змінної починається з подвійного дефісу - за яким слідує її ім'я в довільній нотації. Наприклад, brand-color або brandColor - це валідні імена змінної, головне, щоб усі змінні в проекті були в однаковій нотації. Ім'я та значення поділяється двокрапкою, а в кінці оголошення ставиться крапка з комою.

Цікаво

Оголошення змінної це створення нової CSS-властивості, яка успадковується і тому доступна скрізь, оскільки будь-який елемент це нащадок елемента <html>.

Використання змінної

Для отримання доступу до значення змінної використовується вбудована функція var(), якій першим аргументом передається ім'я змінної. 

                        
            :root {
                --brand-color: #3f51b5;
            }
                              
            .section {
                /* Інші стилі */
                border: 2px solid var(--brand-color);
            }
                              
            .section-title {
                /* Інші стилі */
                color: var(--brand-color);
            }
                              
            .section-title::before {
                /* Інші стилі */
                background-color: var(--brand-color);
            }

Кода стало не набагато більше, але він більш гнучкий та читабельний. Якщо необхідно змінити колір бренду, достатньо замінити значення змінної --brand-color і колір зміниться скрізь, де вона використовується.

Корисно

CSS-змінні застосовуються не тільки для зберігання палітри кольорів проекту, в них можна зберігати будь-що. Наприклад, стандартні розміри шрифту, радіус заокруглення рамок, загальні відступи та поля, ширини контейнера, точки перелому для адаптивної верстки тощо. Тобто будь-які значення, які використовуються повторно в різних CSS-правилах.

Резервні значення

Функція var(ім'я_змінної, фолбек) може приймати два значення. Другий аргумент це фолбек - значення, яке використовуватиметься якщо змінна, передана першим аргументом, не визначена. 

                        
            .section {
                border: 2px solid var(--brand-color, orange);
            }
                              
            .section-title {
                color: var(--brand-color, black);
            }
                              
            .section-title::before {
                background-color: var(--brand-color, tomato);
            }

Корисно

Можливість передати значення за замовчуванням це насамперед можливість, а не обов'язкова вимога під час використання CSS-змінних. Немає потреби скрізь вказувати фолбек.

Область видимості

Змінна --brand-color глобальна, тому що визначена в root, тобто доступна в будь-якому іншому елементі. Якщо значення використовуватиметься лише у певній частині сторінки, можна створювати змінні з локальною областю видимості. Вони будуть доступні тільки для того елемента та його нащадків, у якому були оголошені. 

                        
            <div class="alert">
                <p class="alert-text">Alert text</p>
                <svg class="alert-icon"></svg>
            </div>
